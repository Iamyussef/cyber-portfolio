**Linux Fundamentals Part 3 — TryHackMe**

**Platform:** TryHackMe  
**Status:** Completed ✅  

Just wrapped up Part 3 of the Linux Fundamentals series on TryHackMe. This one felt a bit different compared to the first two. Before, it was more about learning how to move around and not get lost in the terminal. This time, it felt like actually *using* Linux.

A big part of it focused on processes. Commands like `ps`, `top`, and `kill` made more sense now — it’s like you’re finally seeing what’s going on behind the scenes instead of just typing commands blindly. When cron jobs came into the picture, it clicked why so many things in Linux run automatically without you even noticing.

Permissions weren’t completely new, but going over `chmod` and `chown` again helped reinforce things. Looking at it from a security perspective made it more interesting, too. Misconfigured permissions show up a lot in CTFs, so it’s something worth being comfortable with.

Also spent some time on file transfers using `wget` and `scp`. Not the most exciting topic, but honestly, it’s one of those things you’ll need at some point. Same with text editors, I usually avoid working in the terminal and just edit files outside, but this pushed me to actually use `nano` properly.

One part I didn’t expect to find that useful was log files. Going through `/var/log` and seeing what kind of information is stored there gave me a better idea of how systems keep track of activity. I can see how that becomes really important later on, especially for troubleshooting or security.

Overall, a solid room. It’s starting to feel less like “learning Linux” and more like just getting used to it, which is probably the goal anyway.
